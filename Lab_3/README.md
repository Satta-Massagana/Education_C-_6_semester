# Синхронизация потоков в .NET (lock vs Monitor)

## Цели
- **Гонки данных** без синхронизации
- **Разница** между `lock` и `Monitor`
- **Deadlock** и его предотвращение
- **Производительность** различных подходов

## Результаты тестирования
```
Количество транзакций: 10000
Ожидаемый итоговый баланс: 85 698,90 ₽

=== Сравнение подходов к синхронизации ===
Без синхронизации: 1012,40 мс, результат: некорректный
С использованием lock: 1033,65 мс, результат: корректный
С использованием Monitor: 1045,88 мс, результат: корректный

Без синхронизации:
  Время: 1012,40 мс
  Итоговый баланс: 82 868,89 ₽
  Корректность: Нет
  Гонки данных: Да

С использованием lock:
  Время: 1033,65 мс
  Итоговый баланс: 85 698,90 ₽
  Корректность: Да
  Накладные расходы: 2,10%

С использованием Monitor:
  Время: 1045,88 мс
  Итоговый баланс: 85 698,90 ₽
  Корректность: Да
  Накладные расходы: 3,31%

Сравнение производительности:
  Ускорение lock vs Monitor: 0,99x
  Накладные расходы lock: 2,10%
  Накладные расходы Monitor: 3,31%

=== Демонстрация deadlock ===
Deadlock не произошёл (это маловероятно).
```
