# Lab 13: распределенная система сжатия данных

Blazor-приложение для параллельного и распределенного сжатия файлов. Система запускает мастер-узел и 4 локальных TCP-воркера, показывает прогресс по каждому воркеру, измеряет производительность и сохраняет результат в папку `CompressedFiles`.

## Возможности

- загрузка файла через одну Blazor-страницу;
- выбор алгоритма: GZip, Deflate, Brotli;
- режимы распределения:
  - один воркер;
  - разделение файла на части;
  - отправка полного файла всем воркерам для сравнения;
- TCP-взаимодействие мастер ↔ воркеры;
- heartbeat-мониторинг состояния узлов;
- переназначение задачи при отказе воркера;
- прогресс-бары по воркерам;
- статистика CPU, сетевого обмена и общего времени;
- отмена задачи через `CancellationToken`;
- сохранение результата в `CompressedFiles`.

## Запуск

```
dotnet restore
dotnet run
```

После запуска откройте адрес из вывода ASP.NET Core, например:

```text
http://localhost:5000
```

## Как пользоваться

1. Выберите файл.
2. Выберите алгоритм сжатия.
3. Выберите режим распределения.
4. Нажмите `Сжать файл`.
5. Следите за прогрессом по каждому воркеру.
6. После завершения скачайте файл по ссылке или откройте папку `CompressedFiles`.

## Архитектура

Проект использует мастер-рабочую архитектуру:

- мастер принимает файл, запускает бенчмарк, выбирает воркеров и агрегирует результат;
- воркеры слушают TCP-порты `5101-5104`;
- монитор отправляет heartbeat каждые 2 секунды;
- Dataflow-пайплайн параллельно отправляет задания воркерам.

Подробности описаны в `SystemArchitecture.md`.

## Производительность

Для файла 32 MB в контрольном сценарии режим разделения на 4 части показывает примерно 2.5-3x ускорение относительно последовательного сжатия. Подробная методика и таблицы приведены в `PerformanceReport.md`.

## Важные файлы

- `Program.cs` — точка входа.
- `MainPage.razor` — веб-интерфейс.
- `MasterCompressionService.cs` — логика мастера.
- `WorkerNodeServer.cs` — логика воркера.
- `CompressionPipeline.cs` — TPL Dataflow.
- `WorkerRegistry.cs` — heartbeat, состояние и балансировка.
- `ImplementationGuide.md` — руководство по реализации.
- `SystemArchitecture.md` — архитектурное описание.
- `PerformanceReport.md` — отчет о производительности.

## Ограничения

Режим разделения сохраняет результат в формате `.dcmp`, потому что части файла сжимаются независимо. Для обычных одиночных результатов используются стандартные расширения `.gz`, `.deflate` и `.br`.
